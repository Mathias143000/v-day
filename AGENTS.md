# AGENTS.md

## Project Direction

This is a small interactive Valentine page. Keep it lightweight: plain HTML/CSS plus TypeScript is a better fit than a Vue rewrite unless the app grows into multiple complex screens.

Current structure:

- `index.html` is the question screen.
- `yes.html` is the celebration screen.
- `style.css` contains shared layout, responsive styles, and animations.
- `src/script.ts` controls the question screen.
- `src/yes-script.ts` controls the celebration screen.
- `script.js` and `yes-script.js` are generated by `npm run build` and are loaded by the HTML files.

## Priority Improvements

1. Keep source files UTF-8.
   - Russian copy and emoji should render correctly in the browser.
   - Edit TypeScript sources first, then run `npm run build`.

2. Improve animation polish without adding a framework.
   - Keep GIF fades smooth.
   - Use CSS variables for animation timing.
   - Add `prefers-reduced-motion` support for wiggle, bounce, hearts, GIF fade, runaway movement, and confetti.
   - Prefer transform-based animation where it avoids layout shifts.

3. Improve responsiveness.
   - Test at `320`, `375`, `430`, `768`, `1024`, and desktop widescreen widths.
   - Allow mobile vertical scrolling when dynamic button text grows.
   - Use `svh` or `dvh` for mobile viewport height where useful.
   - Keep the dynamic "Да" button from covering the GIF, message, or music toggle.

## TypeScript Notes

- Keep browser-facing functions attached to `window` while HTML uses inline handlers.
- Prefer typed DOM lookups through a small helper instead of repeated nullable casts.
- Keep shared behavior small before introducing abstractions.
- If duplication between screens grows, extract shared music helpers into `src/music.ts`.

## Accessibility Notes

- Keep `aria-live` for tease messages.
- Keep music toggle `aria-label` values accurate for both states.
- Do not assume autoplay always works; keep the first-interaction fallback.
- Ensure all buttons remain keyboard reachable and have visible focus states.

## Validation Checklist

After edits:

- Run `npm run typecheck`.
- Run `npm run build`.
- Open `index.html` and `yes.html` in a browser.
- Verify there are no console errors.
- Test repeated "Нет" clicks on mobile and desktop.
- Test the music toggle on both screens.
- Test reduced-motion mode when animation behavior changes.

## Implementation Guardrails

- Keep the playful Valentine behavior intact.
- Do not add Vue, routing, deployment changes, CI changes, or new infrastructure unless explicitly requested.
- Keep commits focused and avoid unrelated refactors.
